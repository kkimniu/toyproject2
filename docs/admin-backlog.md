# Admin Page Backlog

## Goal

관리자 페이지를 한 번에 크게 만들지 않고, 리뷰 가능한 작은 PR 단위로 나누어 순차적으로 구현한다.

현재 프로젝트에는 다음 기반만 존재한다.

- `ADMIN` 권한
- `/api/admin/**` 관리자 전용 보안 규칙
- 비어 있는 `AdminRestController`

아직 없는 것은 관리자 화면 진입점, 관리자 전용 JSP, 실제 관리자 기능이다.

## Recommended Order

1. 관리자 페이지 기본 진입 구조 추가
2. 관리자 회원 목록 조회 API 추가
3. 관리자 회원 목록 화면 추가
4. 회원 정지/해제 기능 추가
5. 관리자 신고 목록 조회 API 추가
6. 관리자 신고 목록 화면 추가
7. 신고 처리 기능 추가

## Issue 1

### Title

관리자 페이지 기본 진입 구조 추가

### Scope

- 관리자 전용 화면 라우팅 추가
  - 예: `/admin`
- 관리자 메인 JSP 화면 추가
  - 초기에는 빈 대시보드 또는 기본 레이아웃만 구성
- 관리자 권한 사용자만 관리자 화면에 접근 가능하도록 설정
- 로그인한 사용자가 관리자일 경우 헤더에 `관리자` 진입 메뉴 노출

### Acceptance Criteria

- 일반 사용자는 관리자 페이지에 접근할 수 없다.
- 관리자 계정은 `/admin` 페이지에 접근할 수 있다.
- 관리자 계정으로 로그인했을 때 헤더에서 관리자 페이지로 이동할 수 있다.
- 이후 회원 관리, 신고 관리 기능을 붙일 수 있는 기본 화면 구조가 마련된다.

## Later Issue Candidates

### Issue 2

`관리자 회원 목록 조회 API 추가`

- 관리자만 회원 목록을 조회할 수 있다.
- 목록 응답에는 최소한 회원 ID, 이메일, 이름, 권한, 상태, 가입일이 포함된다.
- 향후 검색, 필터, 페이지네이션을 붙일 수 있는 응답 구조를 고려한다.

### Issue 3

`관리자 회원 목록 화면 추가`

- 관리자 페이지에서 회원 목록을 볼 수 있다.
- 기본 테이블 레이아웃과 빈 상태를 제공한다.
- Issue 2의 API와 연결한다.

### Issue 4

`관리자 회원 정지 및 해제 기능 추가`

- 관리자는 일반 회원을 정지 또는 해제할 수 있다.
- 본인 계정과 다른 관리자 계정에 대한 처리 정책을 명확히 한다.
- 처리 후 회원 목록 상태가 즉시 반영된다.

### Issue 5

`관리자 신고 목록 조회 API 추가`

- 관리자만 신고 목록을 조회할 수 있다.
- 신고 대상, 신고자, 사유, 생성일, 처리 상태를 확인할 수 있다.

### Issue 6

`관리자 신고 목록 화면 추가`

- 관리자 페이지에서 신고 목록을 볼 수 있다.
- 처리 상태별로 확인할 수 있는 기본 UI를 제공한다.

### Issue 7

`관리자 신고 처리 기능 추가`

- 관리자는 신고를 처리 완료 상태로 변경할 수 있다.
- 처리 결과가 목록에 반영된다.

## PR Slicing Rule

- 한 PR에는 하나의 사용자 가치만 담는다.
- 화면, API, 상태 변경 로직이 모두 필요한 기능이라도 가능하면 순서를 나누어 붙인다.
- 리뷰어가 기능 하나를 독립적으로 확인할 수 있을 정도의 크기를 유지한다.
