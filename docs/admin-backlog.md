# Admin Page Backlog

## Current Status

관리자 1차 기능은 아래 순서로 구현을 마쳤다.

1. 관리자 페이지 기본 진입 구조
2. 관리자 회원 목록 조회 API
3. 관리자 회원 목록 화면
4. 관리자 회원 정지 및 해제
5. 관리자 신고 목록 조회 API
6. 관리자 신고 목록 화면
7. 관리자 신고 처리 기능

현재 가능한 일:

- 관리자 페이지 진입
- 회원 목록 조회
- 일반 회원 정지 및 해제
- 신고 목록 조회
- 신고 상태별 확인
- 신고를 `PENDING`에서 `RESOLVED`로 처리 완료

## Next Recommended Issue

### Title

관리자 신고 처리 상세 정보 추가

### Why

현재 신고 처리는 `RESOLVED` 여부만 남는다.

실제 운영 기능으로 쓰려면 다음 정보가 필요하다.

- 어떤 결과로 처리했는지
- 신고자에게 어떤 안내를 남겼는지
- 어떤 관리자가 처리했는지
- 언제 처리했는지

이 정보가 있어야 운영 이력을 추적할 수 있고, 이후 신고자에게 처리 결과를 보여주는 기능으로도 자연스럽게 확장할 수 있다.

### Scope

- 신고 처리 결과 필드 추가
  - 예: `ACCEPTED`, `REJECTED`, `NO_ACTION`
- 신고자 안내 문구 저장
- 처리 관리자 저장
- 처리 시각 저장
- 관리자 신고 처리 API 확장
- 신고 목록에서 처리 결과 확인 가능

### Suggested DB Changes

- `report.resolution_type`
- `report.resolution_message`
- `report.processed_by`
- `report.processed_at`

### Acceptance Criteria

- 관리자는 신고 처리 시 처리 결과를 선택할 수 있다.
- 관리자는 신고자에게 전달할 안내 문구를 입력할 수 있다.
- 처리한 관리자와 처리 시각이 저장된다.
- 신고 목록에서 처리 결과를 확인할 수 있다.
- 이미 처리된 신고는 다시 처리할 수 없다.
