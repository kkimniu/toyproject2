## Slicing Rule

- 한 작업은 사용자가 직접 확인할 수 있는 하나의 운영 가치 단위로 묶는다.
- 화면, API, 상태 변경 로직이 함께 필요한 기능은 가능하면 같은 작업에서 끝낸다.
- 독립적으로 테스트할 수 없을 정도로 작게 나누지 않는다.
