## Remaining Work

### Issue Candidate 1

#### Title

상위 관리자 회원 탈퇴 처리 기능 추가

#### Why

반복 위반 회원에 대한 최종 제재 수단이 현재 관리자 화면에는 없다.

#### Scope

- 기존 회원 탈퇴와 같은 soft delete 흐름 재사용
- 상위 관리자만 탈퇴 처리 가능
- 본인 계정, 상위 관리자, 이미 삭제된 회원은 처리 불가
- 관리자 작업 로그에 회원 탈퇴 이력 저장

#### Acceptance Criteria

- 상위 관리자만 회원 탈퇴 처리를 수행할 수 있다.
- 탈퇴 처리는 실제 삭제가 아니라 기존 soft delete 정책을 따른다.
- 일반 관리자는 탈퇴 처리 버튼을 볼 수 없거나 실행할 수 없다.
- 본인 계정, 상위 관리자, 이미 삭제된 회원은 탈퇴 처리할 수 없다.
- 탈퇴 처리 이력이 관리자 작업 로그에 남는다.

### Issue Candidate 2

#### Title

관리자 작업 로그 대상 상세 진입 추가

#### Why

작업 로그에서 대상 ID만 보이면 실제 회원이나 신고 내용을 다시 찾는 데 시간이 든다.

#### Scope

- 회원 대상 로그에서 해당 회원 정보로 바로 이동
- 신고 대상 로그에서 해당 신고 정보로 바로 이동
- 대상이 삭제되었거나 접근할 수 없을 때의 표시 처리

#### Acceptance Criteria

- 관리자는 작업 로그에서 대상 회원 또는 신고로 바로 이동할 수 있다.
- 접근할 수 없는 대상은 깨진 링크 대신 명확한 상태로 표시된다.
- 로그 조회 흐름을 벗어나지 않고 다시 목록으로 돌아올 수 있다.
