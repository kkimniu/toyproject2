## Remaining Work

### Issue Candidate 1

#### Title

관리자 회원 목록 검색 및 필터 고도화

#### Why

현재 회원 목록은 전체 조회만 가능해서 회원 수가 늘어나면 운영 효율이 떨어진다.

#### Scope

- 이름 또는 이메일 검색
- 회원 상태 필터
- 권한 필터
- 가입 기간 조건 검색
- 서버 기준 필터와 페이지네이션 지원

#### Acceptance Criteria

- 관리자는 이름 또는 이메일로 회원을 검색할 수 있다.
- 관리자는 회원 상태와 권한으로 회원 목록을 필터링할 수 있다.
- 관리자는 가입 기간 조건으로 회원을 검색할 수 있다.
- 검색 조건은 서버 요청 기준으로 적용된다.
- 페이지네이션과 함께 사용해도 결과가 일관된다.

### Issue Candidate 2

#### Title

관리자 작업 로그 대상 상세 진입 추가

#### Why

작업 로그에서 대상 ID만 보이면 실제 회원이나 신고 내용을 다시 찾는 데 시간이 든다.

#### Scope

- 회원 대상 로그에서 해당 회원 정보로 바로 이동
- 신고 대상 로그에서 해당 신고 정보로 바로 이동
- 대상이 삭제되었거나 접근할 수 없을 때의 표시 처리

#### Acceptance Criteria

- 관리자는 작업 로그에서 대상 회원 또는 신고로 바로 이동할 수 있다.
- 접근할 수 없는 대상은 깨진 링크 대신 명확한 상태로 표시된다.
- 로그 조회 흐름을 벗어나지 않고 다시 목록으로 돌아올 수 있다.
